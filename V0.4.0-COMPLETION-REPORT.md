**Format String Features:**
```
{VAR}                    → Simple substitution
{VAR:upper}              → Uppercase (also :lower, :uppercase, :lowercase)
{VAR:hex}                → Hex encoding (ABC → 0x414243)
{VAR:len}                → String length
{VAR:trim}               → Remove whitespace  
{VAR:reverse}            → Reverse string
{VAR:pad:10}     → Pad to 10 chars (right-align)
{VAR:left:10}    → Left-align to 10 chars
{VAR:right:10}   → Right-align to 10 chars
{VAR:center:10}  → Center-align to 10 chars
{VAR:upper:left:10}      → Chain specs (composed)
```

**Escape Sequences:**
- `\n` → newline
- `\t` → tab
- `\r` → carriage return
- `\\` → backslash
- `\{` → literal `{`
- `\}` → literal `}`

**Variable Source:** `scope_manager.flatten()` (all variables from current scope)

## Known Limitations (By Design)

1. **Parentheses in Conditions:** Not supported in v0.4.0 (use De Morgan's laws)
2. **Variable Types:** All strings internally; numeric comparison converts to i64
3. **Format Specs:** Simple operations only; complex math deferred to v0.5.0
4. **Lint Rules:** Basic validation; full semantic analysis in v0.4.1
5. **Break/Continue:** Not in v0.4.0 (requires IR layer changes; targeted for v0.4.1)
